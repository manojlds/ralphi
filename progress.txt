## Codebase Patterns
- Tools are registered in `extensions/ralphi/src/tools.ts` via `registerTools()` тАФ each tool uses TypeBox schemas for parameters
- Tool logic should be extracted into separate files (e.g. `ask-tool.ts`) and imported by `tools.ts` for testability
- Mock-based tests use factories from `extensions/ralphi/test/factories/pi.ts` тАФ `createMockUi()` accepts arrays of pre-queued responses for `confirm`, `input`, and `select`
- Pi UI API: `select()` returns `string | undefined` (single-select only), `input()` returns `string | undefined`, `confirm()` returns `boolean` тАФ no native multi-select
- Multi-select is implemented via iterative `select()` calls with a "Done" sentinel option
- All quality checks via `npm run check` (lint + typecheck + tests); pre-commit hook enforces this
- Headless/non-interactive fallback pattern: tools that need UI should accept optional pre-supplied params (`providedAnswers`) and validate them; fail with actionable error + JSON example when neither UI nor pre-supplied params are available
- Skill-contract testing pattern: for prompt/skill-only behavior changes, add unit tests that read the relevant `skills/**/SKILL.md` file and assert required protocol markers/format fields to prevent accidental prompt regressions
- Project-local loop guidance pattern: store iteration preferences at `.ralphi/loop-guidance.md`, manage via `/ralphi-loop-guidance-*` commands, and inject only from `handleBeforeAgentStart` when phase is `ralphi-loop-iteration`
- Optional advanced loop controls pattern: use additive front matter in `.ralphi/loop-guidance.md` (`reviewPasses`, `trajectoryGuard`) so stricter gates are opt-in while default behavior remains unchanged
- Loop config mutation safety pattern: when command helpers rewrite `.ralphi/config.yaml` `loop:` blocks, always preserve additive `loop.*` keys (for example `reflectEvery`, `reflectInstructions`) to avoid silently dropping unrelated settings
- Rollout-safety documentation pattern: capture phase sequencing + go/no-go + rollback policy in `tasks/*.md` and guard it with markdown contract tests under `extensions/ralphi/test`.
